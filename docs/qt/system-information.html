System information tab

The system information tab contains information about your system. These contain the following information.

Contents

  • Os Information
  • Memory Information
  • CPU Information
  • Timing Information
  • Topology

Os Information

This group contains the Operating System information. More specifically:

Category
The general category of the operating system, for instance 'Linux', 'Windows' etc.
Name
The name of the operating system, for instance 'Ubuntu', 'Windows 10' etc.
Version
The version of the operating system, for instance '6.11.0', '2004' etc.

Memory Information

This group contains the main memory (RAM) information. More specifically:

Total size
The main memory size in IEC units.
Page size
The page size that is used in IEC units.
Version
The version of the operating system, for instance '6.11.0', '2004' etc.

CPU Information

This group contains the CPU information. More specifically:

Architecture
The CPU architecture used by Elpida. Note that this does not imply the CPU architecture is this. For instance, you can run the 'i686' version of Elpida on an 'x86-64' cpu, but Elpida will report 'i686' as architecture.
Vendor
The CPU vendor name.
Model name
The CPU model name.

Timing information

This group contains information about timings calculated by Elpida. These timing are used by elpida to precisely calculate time for benchmarks. More specifically:

now() overhead
The average time a call to 'now()' takes on this machine. 'now()' returns a precise time-point.
Loop overhead
The average time an empty loop takes.
B. Ips
Or Bogus instructions per second. This is directly related to 'Loop overhead' and it is the rate of empty loops that the system can run.

Topology

This is the topology of the system. This graph describes the cpu cores/caches/packages structure tree and sizes of each node. From here you can select which cores/threads the custom benchmarks will use. You can select/deselect cores/threads by clicking on them or by massively select them by clicking an ancestor group (for instance clicking on the LD2 cache name will select all cores/threads inside that cache).

  • Numbers in parentheses (e.g. '(2)'), is the id/index the operating system gave to this node.
  • Numbers with a 'iB' (e.g. '32 KiB') suffix is the size of the node in bytes in case the node has memory.
  • Numbers in brackets (e.g. '[1]') is the efficiency this specific core reports. This number makes sense when the cpu has not a standard size of cores, for instance big.LITTLE cpus.
